# 設計書レビュー

指定された設計書ドキュメントを**CLAUDE.mdの設計書作成ガイドライン**に基づいてレビューし、改善提案を行ってください。

## レビュー観点

以下の観点で評価してください：

### 1. コード例の適切性
- [ ] コード例は5-15行以内に収まっているか
- [ ] 完全な実装コードが含まれていないか
- [ ] インターフェース定義や型定義に絞られているか
- [ ] コメントで意図が説明されているか

### 2. 構成の適切性
- [ ] 概要と目的が明確か
- [ ] 責務と役割が定義されているか
- [ ] 図や表で視覚化されているか
- [ ] 適切な順序で説明されているか

### 3. 説明の重点
- [ ] 「なぜ」この設計にしたのか説明されているか
- [ ] トレードオフや代替案が考慮されているか
- [ ] 実装の詳細に偏りすぎていないか

### 4. ドキュメントの粒度
- [ ] ドキュメント種類に応じた適切な詳細度か
- [ ] 「何を」が明確に説明されているか

## 出力形式

以下の形式でレビュー結果を出力してください：

```markdown
## レビュー結果

### ✅ 良い点
- [具体的な良い点を箇条書き]

### ⚠️ 改善提案
1. **[問題点のタイトル]**
   - 現状: [現在の問題]
   - 提案: [改善方法]
   - 理由: [なぜ改善すべきか]

### 📊 評価スコア
- コード例の適切性: [1-5]
- 構成の適切性: [1-5]
- 説明の重点: [1-5]
- 総合評価: [1-5]

### 💡 具体的な修正例
[具体的な修正例を示す]
```

## 使用方法

```bash
# 特定の設計書をレビュー
/review-design-doc docs/design/backend/01-architecture.md

# または、ファイルを指定せずに現在開いているファイルをレビュー
/review-design-doc
```
